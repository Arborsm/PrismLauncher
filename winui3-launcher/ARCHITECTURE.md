# Prism Launcher WinUI 3 架构设计

## 概述

本项目使用现代Windows UI框架(WinUI 3)重新实现Prism Launcher的用户界面，通过native调用桥接层访问原有的Qt/C++核心功能。

## 架构层次

### 1. 用户界面层 (WinUI 3)

**文件**: `src/App.xaml`, `src/MainWindow.xaml`

- **App.xaml**: 应用程序定义和全局资源
- **MainWindow.xaml**: 主窗口UI布局
  - 侧边栏：实例列表、分组筛选
  - 主内容区：实例详情、快速操作
  - 状态栏：运行状态、统计信息

**特性**:
- 现代化Windows 11设计风格
- 响应式布局，支持不同屏幕尺寸
- 深色主题支持
- 流畅的动画和过渡效果

### 2. 桥接层 (C++/WinRT)

**文件**: `src/bridge/`

#### PrismLauncherBridge
- 主要的桥接类，负责协调UI和核心功能
- 提供高级API供UI层调用
- 管理事件回调和状态同步

#### InstanceWrapper
- 包装单个实例对象
- 提供类型安全的访问接口
- 处理Qt和WinRT之间的数据转换

#### InstanceListWrapper
- 包装实例列表管理功能
- 提供批量操作接口
- 管理分组和筛选逻辑

### 3. 核心包装层

**文件**: `src/core/PrismCore.cpp`

- 封装Prism Launcher的核心库
- 提供简化的初始化接口
- 隔离Qt依赖，便于维护

### 4. 原生核心库 (Qt/C++)

**原有Prism Launcher核心**:
- `Application`: 主应用程序类
- `InstanceList`: 实例列表管理
- `BaseInstance`: 实例基类
- `LaunchController`: 启动控制器

## 数据流

```
用户操作 → WinUI 3 UI → C++/WinRT桥接 → Qt核心库 → 系统API
                ↓
        事件回调 ← 状态更新 ← 数据转换 ← 业务逻辑
```

## 关键技术

### 1. C++/WinRT
- 使用C++/WinRT进行WinRT和C++之间的互操作
- 类型安全的接口设计
- 自动内存管理

### 2. 数据转换
- QString ↔ std::wstring
- Qt信号槽 ↔ WinRT事件
- Qt对象 ↔ WinRT对象

### 3. 异步操作
- 使用WinRT的异步模式
- 避免UI线程阻塞
- 提供进度反馈
